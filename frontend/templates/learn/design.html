def withdraw(self, amount: int) -> int:
    if amount > self._balance:
        raise InsufficientFundsError(
            "Savings account cannot go below 0."
        )
    self._balance -= amount
    return self._balance

CurrentAccount

backend/domain/current_account.py
Overrides:
  • withdraw() - Overdraft allowed (balance ≥ -500)
  • account_type - Returns "current"
OVERDRAFT_LIMIT = 500

def withdraw(self, amount: int) -> int:
    new_balance = self._balance - amount
    if new_balance < -self.OVERDRAFT_LIMIT:
        raise InsufficientFundsError(
            f"Overdraft limit exceeded."
        )
    self._balance = new_balance
    return self._balance
Inheritance Polymorphism

💡 Key Insight: Polymorphism in Action

When you call account.withdraw(100), Python automatically uses the correct implementation:

  • If account is a SavingsAccount → checks balance ≥ 0
  • If account is a CurrentAccount → checks balance ≥ -500

This is polymorphism: same method name, different behavior based on the object's actual type.
