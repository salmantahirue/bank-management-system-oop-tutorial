OOP Concepts in Action

Where exactly did we use OOP in this project?

🔒

Encapsulation

What: Hiding internal state.

Where: Account._balance is protected. You cannot change it directly; you MUST use deposit() or withdraw().

Click to learn more →
🧬

Inheritance

What: Sharing common logic.

Where: SavingsAccount and CurrentAccount both inherit from Account to get shared attributes like account_id and deposit().

Click to learn more →
🎭

Polymorphism

What: Same method, different behavior.

Where: calling account.withdraw(100) works differently depending on whether it's a Savings or Current account.

Click to learn more →
👻

Abstraction

What: Enforcing a contract.

Where: The Account class is an Abstract Base Class (ABC). You cannot create a plain Account; you must create a specific type.
