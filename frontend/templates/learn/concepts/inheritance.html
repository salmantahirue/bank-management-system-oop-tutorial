🧬 Inheritance

Reusing code by creating new classes based on existing ones.

What is Inheritance?

Inheritance allows a class (child/subclass) to inherit attributes and methods from another class (parent/superclass). This promotes code reuse and establishes an "is-a" relationship.

Real-World Analogy: A SavingsAccount IS AN Account. It has all the properties of an account (balance, account_id) plus its own specific rules.

Our Inheritance Hierarchy

Account (Parent/Base Class)
▼ Inherited by
SavingsAccount Inherits: account_id, user_id, _balance, deposit()
CurrentAccount Inherits: account_id, user_id, _balance, deposit()

Code Example: SavingsAccount Inherits from Account

📄 backend/domain/savings_account.py 
from backend.domain.account import Account

class SavingsAccount(Account):
    """
    SavingsAccount inherits from Account.
    It automatically gets:
    - account_id, user_id, _balance (from __init__)
    - deposit() method
    - balance property
    """
    
    @property
    def account_type(self) -> str:
        return "savings"
    
    def withdraw(self, amount: int) -> int:
        # ✅ Can use parent's helper methods
        self._require_positive_amount(amount)
        
        # Savings-specific logic: no overdraft
        if amount > self._balance:
            raise InsufficientFundsError(
                "Savings account cannot go below 0."
            )
        
        # ✅ Can access parent's protected attribute
        self._balance -= amount
        return self._balance

What Gets Inherited?

✅ Inherited Attributes:
  • account_id - Set in parent's __init__
  • user_id - Set in parent's __init__
  • _balance - Protected, accessible to subclasses
✅ Inherited Methods:
  • deposit() - Works as-is, no need to rewrite
  • balance property - Read-only access
  • _require_positive_amount() - Helper method
🔄 Overridden Methods:
  • withdraw() - Each subclass implements differently
  • account_type - Returns "savings" or "current"

Benefits of Inheritance

📦 Code Reuse

We don't duplicate deposit() logic in both SavingsAccount and CurrentAccount.

🔧 Easy Maintenance

If we change deposit() in Account, both subclasses automatically get the update.

🎯 Consistency

All account types share the same interface, making the code predictable.

🎯 Key Takeaway

Inheritance lets us write common code once in the parent class and reuse it in child classes. This follows the DRY (Don't Repeat Yourself) principle.
